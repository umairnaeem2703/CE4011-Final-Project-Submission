\section{Limitations and Future Work}

The final submitted desktop MVP is intentionally limited to Static Analysis and Modal Analysis. RSA and THA backend work may remain in the repository for future extension, but those workflows are not presented as submitted desktop features.

Current limitations include the two-dimensional model scope, educational lumped-mass default for modal workflows, limited polished report automation, and remaining need for final screenshot capture and final numerical comparison tables in the submission documents. The solver core should remain transparent and standard-library oriented unless an explicit future dependency decision is approved.

Future work may include promoted RSA/THA desktop workflows, expanded classroom manuals, richer report/PDF export automation, additional validation examples, improved installer packaging, and broader visualization options. Any future analysis feature should follow the same model-to-assembly-to-solver-to-result-object-to-visualization architecture rather than moving solver logic into UI code.
