\section{Result Visualization and Post-Processing}

Post-processing is provided through Tkinter result windows and matplotlib plots. Plotting functions consume a model plus a result object; they do not perform solver math. This separation keeps plotted values traceable to \codeterm{StaticResults}, \codeterm{ModalResults}, and the post-processor output used by tests and exports.

The principal visual outputs are model preview, static deformed shape, axial force diagram, shear force diagram, bending moment diagram, and modal mode shapes. The numerical counterpart is available in result tables: matrices, vectors, DOF labels, displacements, reactions, member forces, member diagram values, eigenvalues, frequencies, periods, mode-shape components, and participation quantities.

This post-processing design supports the educational goal of pairing a visual result with the exact numerical data behind it. For example, a static deformed shape can be checked against the displacement table, and an N/V/M plot can be traced back to member-end forces and load recovery data.
