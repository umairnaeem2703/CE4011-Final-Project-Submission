\section{Analysis Procedure}

\subsection{Static Analysis Workflow}

Static analysis begins only after a \codeterm{StructuralModel} exists and validation succeeds. The desktop UI collects user input, but the numerical path is performed by model, assembly, solver, and post-processing modules:

\begin{enumerate}
\item The desktop command obtains the current model from \codeterm{ModelBuilder}.
\item \codeterm{StructuralValidator} checks that the model is structurally meaningful.
\item The DOF optimizer assigns equation numbers and identifies free and restrained DOFs.
\item The matrix assembler forms the global stiffness matrix \(K\), force vector \(F\), reduced stiffness matrix \(K_{ff}\), and reduced force vector \(F_f\).
\item The static linear system is solved for free displacements.
\item The full displacement vector is reconstructed and reactions are recovered from \(R=Ku-F\).
\item The post-processor recovers local member-end forces and N/V/M diagram data.
\item A \codeterm{StaticResults} object is passed to result tables, plots, and export helpers.
\end{enumerate}

\subsection{Modal Analysis Workflow}

Modal analysis uses the same input model but adds mass assembly and dynamic DOF handling. The workflow is:

\begin{enumerate}
\item The model is validated and DOFs are numbered.
\item The stiffness assembler builds the full stiffness matrix needed for the dynamic system.
\item The dynamic assembler forms the active dynamic stiffness and mass matrices using the selected mass representation.
\item Dynamic DOFs with mass are retained; disconnected zero-mass DOFs may be removed.
\item Stiffness-coupled massless DOFs are statically condensed before solving the eigenproblem.
\item \codeterm{ModalSolver} solves \(K\phi=\lambda M\phi\), computes frequencies and periods, normalizes mode shapes, and evaluates modal participation quantities.
\item A \codeterm{ModalResults} object is displayed through modal summary tables, matrix views, and mode-shape plots.
\end{enumerate}
