\section{Introduction}

The CE 4011 term project requires a structural analysis software package that includes a computational engine, an organized input workflow, post-processing tools, object-oriented design, and validation against reliable reference solutions. This project satisfies that requirement as a Python desktop minimum viable product for two-dimensional educational structural analysis. The submitted user-facing application is a Tkinter desktop program that lets a student build or load a model, validate the model, run Static or Modal analysis, inspect intermediate numerical data, visualize results, and export selected outputs.

The educational purpose of the software is as important as the final displacement or frequency values. Commercial structural analysis tools often hide the degree-of-freedom map, assembled stiffness matrix, reduced system, force vector, member force recovery, mass matrix, and modal participation calculations. This project intentionally exposes those intermediate values so that students can trace the calculation path from idealized structure to solver result. The report therefore describes not only what the software does, but also how the object model, assembly layer, solvers, result objects, and result viewers preserve engineering meaning.

\reportfigure{Main UI}{main_ui.png}{Final Tkinter desktop window showing the canvas, object tree, property editing area, analysis actions, and result access controls.}

The final submitted desktop scope is Static Analysis and Modal Analysis. The repository may retain Response Spectrum Analysis (RSA) and Time-History Analysis (THA) backend experiments as future-extension work, but those workflows are not claimed as part of the final desktop MVP. Detailed numerical comparison tables and long validation explanations are placed in the verification appendix so that the main report remains focused on the project scope, theory, architecture, workflow, and validation summary.
