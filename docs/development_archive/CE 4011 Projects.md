The objective of the term project is to develop a structural analysis software package that includes both
a computational engine and a user interface for defining the structural model, running the analysis, and
post-processing the results.

The minimum project requirements are as follows:

1.  User Interface for Model Input

The  software  must  include  a  user  interface  that  simplifies  the  structural  model  input  process.  This
interface does not have to be a fully graphical user interface. Spreadsheet-based, script-based, or form-
based input systems are acceptable, provided that they are clear, organized, and easy to use.

You  may  also  use  external  software  such  as  AutoCAD,  Revit,  Grasshopper,  Excel,  or  similar  tools  for
preparing or importing the structural model data. However, the workflow between the external software
and the developed analysis program must be clearly explained.

2.  Computational Engine

The  computational  part  of  the  software  must  include  analysis  capabilities  that  go  beyond  the  topics
directly covered in the assignments. The proposed computational features must be clearly described in
the project proposal.

The project is not limited to two-dimensional linear static analysis. You may choose to work on topics such
as dynamic analysis, simple nonlinear analysis, stability analysis, three-dimensional analysis, or structural
design checks.

3.  Object-Oriented Program Structure

The software must be developed using an object-oriented programming approach. At a minimum, the
computational engine must have an object-oriented structure.

The final report must include static class diagrams and a detailed explanation of the main design decisions.
The report should explain the purpose of the major classes, their responsibilities, and the relationships
between them.

4.  Result Visualization and Post-Processing

The software must include visualization tools for post-processing the analysis results. At a minimum, the
program should provide graphical outputs such as deformed shapes and internal force diagrams.

5.

Installation Package or Installation Manual

You  must  prepare  either  an  installer  or  a  detailed  installation  manual.  The  installation  manual  must
explain, step by step, how the software can be installed and run on another computer.

The manual should include all required software, libraries, packages, and dependencies.

6.  User Manual

You should prepare a user manual explaining how to use the software. The manual should describe the
input format, model creation process, analysis procedure, and interpretation of results.

The user manual should include at least one complete example problem showing the full workflow from
model input to result visualization.

7.  Testing and Validation

The analysis results must be verified using hand calculations, textbook examples, commercial structural
analysis  software,  or  another  reliable  reference  solution.  The  final  report  must  include  a  comparison
between the developed software and the reference results.

The report should include a description of the tests performed to check the correctness of the program.
At least several unit tests or verification examples should be provided for important components such as
nodes, elements, materials, stiffness matrix assembly, boundary conditions, and result recovery.
