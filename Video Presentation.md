Video Presentation

Each student must prepare a video presentation for the term project. The video presentation
should be between 5 and 10 minutes long.

The video may be uploaded to YouTube as an unlisted video, and the video link should be
submitted together with the final project files. The video link must be accessible to the
instructor and teaching assistants.

The video presentation should, at least, include the following parts:

1.  Brief Introduction to the Program

Briefly  introduce  the  purpose  and  scope  of  the  developed  software.  Explain  what  type  of
structural system the program can analyze and summarize its main analysis capabilities.

2.  Program Structure and Data Organization

Explain  the  general  structure  of  the  program.  Describe  the  main  classes,  objects,  data
structures, and input/output organization used in the software.

This  part  should  clearly  show  how  the  object-oriented  structure  was  used  in  the
computational engine.

3.  User Interface and Input Procedure

Demonstrate  how  a  structural  model  is  defined  using  the  program.  Explain  the  input
procedure,  including  geometry  definition,  material  and  section  properties,  boundary
conditions, and loads.

If  external  software  such  as  AutoCAD,  Revit,  Grasshopper,  or  Excel  is  used  for  model
preparation, briefly explain the data transfer workflow.

4.  Analysis Demonstration

Run the analysis of at least one example model during the video. The demonstration should
show the complete workflow from model preparation to execution of the analysis.

The example should not be a hardcoded model. It should demonstrate that the program can
analyze a user-defined structural model within the selected project scope.

5.  Post-Processing and Visualization

Present the analysis results using the visualization tools developed in the project. The video
should  show  relevant  outputs  such  as  deformed  shape,  internal  force  diagrams,  support
reactions, displacement results, or other result plots depending on the scope of the project.

6.  Verification

Include a separate verification section. Compare the results obtained from the developed
software  with  hand  calculations,  textbook  examples,  commercial  structural  analysis
software, or another reliable reference solution.

The  verification  section  should  clearly  state  the  reference  solution  used  and  should
compare  relevant  results  such  as  displacements,  reactions,  internal  forces,  natural
frequencies, or design check results, depending on the project scope.

7.  Limitations and Possible Improvements

Briefly discuss the current limitations of the software and possible future improvements.

The video should be clear, organized, and easy to follow. The presentation should focus on
the developed software, its computational implementation, its object-oriented structure,
the analysis workflow, and the verification of the results.

Please submit the video link as a text file.
